# 일일 루틴: T+2일자 콘텐츠 생성

이 워크플로우는 일일 콘텐츠 생성을 위한 표준 절차를 정의합니다.
**핵심 규칙:** 특정 날짜(Day T)에 우리는 2일 뒤(Day T+2)의 콘텐츠를 생성합니다.
**표시 규칙:** 웹사이트는 항상 오늘(Day T)의 콘텐츠를 기본으로 표시합니다.

## 1. 목표 날짜 결정 (Determine Target Date)

- `목표 날짜` = `오늘 + 2일`로 계산합니다.
- 예시: **2월 6일**에 실행하는 경우, 목표 날짜는 **2월 8일**입니다.

## 2. 콘텐츠 생성 (Generate Content) — 🍒 체리

- **역할:** 체리는 30년차 수능 영어 출제 위원입니다.
- **중요 규칙:** `.agent/prompts/csat_rules.md`에 정의된 혼합 언어 규칙을 준수하십시오. 질문은 한국어, 선택지는 유형에 따라 한국어/영어로 작성해야 합니다.
- **인증 규칙:** NotebookLM 인증이 만료되거나 실패할 경우, **반드시 사용자에게 인증 갱신을 요청**하십시오. 임의로 데이터를 생성하지 마십시오.

## 2.1 문제 구성 (Total 21 Questions)

- **기초 (Foundation):** 5문제 (듣기/읽기 혼합)
- **중급 (Intermediate):** 5문제 (듣기/읽기 혼합)
- **고급 (Advanced):** 6문제 (고난도 유형, **Prime Killer 1문제 포함**)
- **실전 모의고사 (Mock Exam):** 5문제 (실전 난이도 5문항 세트)

## 3. 🍓 딸기 검증 (Strawberry Review)

**이 단계는 체리가 문제를 모두 생성한 직후 반드시 실행합니다.**

- **역할:** 딸기는 냉철한 수석 검토 위원입니다. 아주 작은 문법 오류나 논리적 비약도 놓치지 않습니다.
- **방법:** 생성된 21문항 전체를 `.agent/prompts/strawberry_review.md`의 5가지 기준으로 독립 검증합니다.
- **5가지 검증 기준:**
  1. 정답 정확성 (Answer Accuracy)
  2. 오답 배타성 (Distractor Exclusivity)
  3. 지문-질문 일관성 (Text-Question Alignment)
  4. 계산 검증 (Calculation Check)
  5. 어법 검증 (Grammar Check)
- **출력:** 각 문항에 대해 다음을 기록합니다:
  - ✅ **통과** — 정답이 올바름
  - ⚠️ **이의 제기** — 정답이 의심됨 (수정 의견 + 근거 포함)

### 3.1 체리 재검토 (Cherry Re-review)

- 딸기가 이의를 제기한 문항에 대해 체리가 재검토합니다.
- **수용:** 딸기의 의견이 타당하면 정답을 수정합니다.
- **거부:** 체리의 원래 정답이 맞다면 반박 근거를 제시하고 원본을 유지합니다.

### 3.2 🤖 Gemini 교차 검증 (Gemini Cross-Verification)

**체리-딸기 검증 후, 반드시 외부 AI(Gemini)를 통한 교차 검증을 수행합니다.**

- **방법:**
  1. 생성된 21문항 전체를 NotebookLM에 **텍스트 소스로 추가**합니다.
  2. NotebookLM(Gemini)에게 각 문항을 **독립적으로 풀어서** 정답을 판단하도록 요청합니다.
  3. Gemini의 판단 정답과 체리의 정답을 비교합니다.
- **불일치 발생 시:**
  - 체리가 Gemini의 근거를 검토하고 수용 또는 거부를 결정합니다.
  - 어법/계산 문제는 Gemini의 판단을 우선 존중합니다.
- **검증 후:** 소스로 추가한 문항 텍스트는 삭제하여 NotebookLM을 깨끗하게 유지합니다.

### 3.3 최종 보고서 제출 (Final Report)

- 3중 검증(체리 + 딸기 + Gemini) 완료 후, 다음 내용을 포함한 리뷰 보고서를 사용자에게 제출합니다:
  - 문항별 3중 검증 결과 테이블 (🍒/🍓/🤖)
  - 이의가 제기된 문항 목록
  - 🍓 딸기의 의견 + 근거
  - 🤖 Gemini의 판단 + 근거
  - 🍒 체리의 최종 판단 (수용/거부 + 이유)
  - 수정된 문항이 있을 경우 변경 내역

## 4. 데이터 파일 생성 (Create Data File)

- 파일 생성: `src/data/daily/{YYYYMMDD}.ts`
- 형식: 엄격한 TypeScript 타입(`import type { DailySet }`)을 준수합니다.
- **주의:** 이 단계에서는 딸기 검증을 통과한 최종 확정 문항만 사용합니다.

## 5. 데이터 등록 (Register Data)

- `src/data/questions.ts` 파일에 새 파일을 import 합니다.
- `dailyContent` 객체에 추가합니다.

## 6. 배포 (Deployment)

- 변경 사항을 `main` 브랜치에 푸시합니다.
- Vercel 배포를 확인합니다.
- **확인:** `https://csat-prep-2027.vercel.app`에 접속하여 확인합니다. 기본적으로 **오늘 자** 콘텐츠가 보여야 하지만, 새로운 **T+2일자** 콘텐츠는 URL `.../practice/{T+2}`를 통해 접근 가능해야 합니다.
